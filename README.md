# パーソナルダイジェスト(PWA)

世界のニュースをあなた好みに学習して届ける、ビルド不要のニュースダイジェストPWAです。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体(HTML/CSS/JSすべて内包) |
| `sw.js` | Service Worker(オフラインキャッシュ・通知) |
| `manifest.json` | PWAマニフェスト |
| `icon-192.png` / `icon-512.png` / `icon-maskable-512.png` / `apple-touch-icon.png` | アイコン |

## 使い始める(ホスティング → iPhoneにインストール)

PWAとして動かすには **HTTPSでの公開が必要** です(Service Workerの仕様)。最も簡単な方法は次のいずれかです。

1. **GitHub Pages**: このフォルダをリポジトリにpushし、Settings → Pages で公開
2. **Netlify Drop** (https://app.netlify.com/drop): フォルダをドラッグ&ドロップするだけ
3. **Cloudflare Pages / Vercel** など

動作を軽く試すだけなら、PC上で `python3 -m http.server` を実行し `http://localhost:8000` でも動きます(localhostはHTTPSなしでもSW可)。`file://` で直接開いた場合、記事の閲覧はできますがSW(オフラインキャッシュ・通知)は無効になります。

### iPhoneへのインストール(ホーム画面追加)

1. **Safari** で公開URLを開く(Chromeなど他ブラウザ不可)
2. 共有ボタン(□↑)をタップ
3. 「**ホーム画面に追加**」を選択
4. ホーム画面のアイコンから起動 → 初回オンボーディングで国・言語・カテゴリ・通知時刻を設定

## 主な機能

- **国の複数選択**: 日本+アメリカなど18か国から複数選択。Google News RSSの国別版(`hl`/`gl`/`ceid`)を利用
- **表示言語の独立選択**: ニュースの言語とUIの言語(日本語/英語)を別々に設定可能。英語学習用に英語ニュース表示も可
- **概要→全文**: カードで見出し+要約、タップで詳細シート(要約・関連報道・元記事リンク)
- **興味学習**: 読んだ記事は+、「✕」でスキップ/「このジャンルを減らす」は−として、カテゴリとキーワードの重みを端末内(IndexedDB)で学習。時間減衰つき。外部送信なし。設定からリセット可
- **朝のダイジェスト**: 指定時刻・指定カテゴリのトップ記事をまとめ、通知+アプリ内バナーで表示
- **ワンステップ成長**: テクノロジー/ビジネス/成長業界(生成AI・半導体・脱炭素など)の検索フィードから重要記事をキュレーションし、上位10件を自動オフライン保存
- **オフライン対応**: アプリ本体・読んだ記事・成長キュレーション・直近フィードをオフライン閲覧可能

## ニュース取得の仕組みと制約(正直な説明)

### 取得経路

Google News RSS(APIキー不要)を情報源とし、ブラウザのCORS制約のため次の順で試行します。

1. **直接fetch** — 通常はCORSでブロックされます(将来プロキシ配下で動かす場合に備えて残置)
2. **rss2json** (`api.rss2json.com`) — **実機ブラウザで動作確認済み**(2026-07-03、日本版トップ/ビジネス、米国版テクノロジー、検索フィードすべて取得成功)
3. **allorigins** (`api.allorigins.win`) — 予備。検証時は応答が不安定でした

参考: 検証時点で `corsproxy.io` は403(登録制に移行)のため採用していません。

### 制約として知っておくべきこと

- **公開プロキシ依存**: rss2jsonは無料の外部サービスであり、レート制限・仕様変更・停止のリスクがあります。アプリ側では15分キャッシュで呼び出しを最小化し、失敗時はキャッシュ表示に自動フォールバックしますが、根本的には「無料での恒久保証はない」経路です。確実性を上げたい場合は、自前のプロキシ(Cloudflare Workersで数十行)を立てて直接fetch経路を活かすのが正攻法です
- **要約の粒度**: Google News RSSのdescriptionは記事本文ではなく、見出し+関連報道リンク集です(検証時: 要約約250字・関連リンク5件)。「全文」は著作権・技術面の制約から元記事リンクで提供します。オフライン時に読めるのは見出し・要約・関連リンクまでです
- **件数**: rss2json無料枠では1フィードあたり10件です。複数国×複数カテゴリを束ねることで一覧の実質件数を確保しています
- **中国のフィード**: Google Newsの中国版(CN)は提供状況が特殊なため、取得できない場合があります。その場合は台湾(繁体字)版の利用をおすすめします

### iOSの通知制約(重要)

- iOS Safariでは、**ホーム画面に追加したPWAのみ**通知APIが使えます(iOS 16.4以降)。ブラウザのタブで開いただけでは通知許可すら要求できません
- **アプリを完全に終了している間の定時ローカル通知は、Web技術では実現できません**。iOSのWeb PushはPushサービス経由(=サーバーが必要)のみで、本アプリはサーバーレス構成のためこれに該当しません
- 本アプリの現実的な動作: (a) アプリが開いている/バックグラウンドにある間はタイマーで指定時刻に通知、(b) 完全終了していた場合は、**次にアプリを開いた瞬間に未読ダイジェストをバナー+通知で表示**します(1日1回)
- 確実な毎朝プッシュが必要になったら、Web Pushサーバー(例: Cloudflare Workers + web-push)を足す拡張が可能です。`sw.js` には push ハンドラを実装済みです
